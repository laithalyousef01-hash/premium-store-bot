# ملخص مشروع Telegram Store Bot – الحالة الحالية

## 1️⃣ فكرة المشروع

المشروع عبارة عن **بوت متجر تلجرام لبيع الاشتراكات والألعاب الرقمية**.

يوفر البوت:

* شراء **اشتراكات جاهزة** مثل:

  * Netflix
  * Spotify
  * ChatGPT Plus
    (الأسعار مخزنة داخل `products.json`)

* طلب **ألعاب رقمية**:

  * العميل يكتب اسم اللعبة
  * الأدمن يحدد السعر يدويًا
  * العميل يقبل السعر
  * العميل يرسل صورة الدفع
  * يتم التسليم

النظام مصمم ليعمل كمتجر رقمي كامل داخل تلجرام.

---

# 2️⃣ بنية المشروع الحالية

المشروع يتكون من الملفات التالية:

```
bot.py
config.py
database.py
products.json
requirements.txt
.env
store.db
```

## bot.py

الملف الرئيسي للبوت ويحتوي:

* أوامر البوت
* منطق الطلبات
* الطلبات الخاصة بالألعاب
* الاشتراكات
* الدفع
* التسليم
* اللغة
* الأزرار
* حماية الإدارة
* logging

## config.py

يحمل إعدادات التشغيل ويقرأها من `.env`.

مثل:

* BOT_TOKEN
* SUPPORT_GROUP_ID
* ORDERS_GROUP_ID
* ADMIN_USER_IDS
* ADMIN_LANG
* بيانات الدفع
* اسم المتجر

## database.py

يدير قاعدة البيانات SQLite ويحتوي:

* إنشاء الجداول
* إنشاء الطلبات
* تحديث الحالة
* حفظ التسليم
* حفظ اللغة

## products.json

يحوي الاشتراكات الجاهزة وأسعارها.

مثال:

```
Netflix
Spotify
ChatGPT Plus
```

## requirements.txt

المكتبات المطلوبة لتشغيل البوت:

```
python-telegram-bot
python-dotenv
```

## .env

يحمل كل الإعدادات السرية:

```
BOT_TOKEN
GROUP_IDS
ADMIN_USER_IDS
LOGGING
PAYMENT_INFO
```

# 3️⃣ ما قمنا بتطويره أثناء العمل

تمت عدة مراحل تحسين قبل الكلاود.

## المرحلة 1 — الحماية الإدارية

تمت إضافة:

* ADMIN_USER_IDS
* منع أي مستخدم غير الأدمن من تنفيذ أزرار الإدارة
* حماية:

```
Send Price
Deliver Game
Deliver Subscription
Not Available
```

---

## المرحلة 2 — Logging احترافي

تمت إضافة:

* تسجيل الأحداث
* تسجيل الأخطاء
* حفظ logs داخل مجلد

مثل:

```
new order
payment received
delivery done
errors
```

هذا مهم جدًا عند تشغيل البوت على الكلاود.

---

## المرحلة 3 — تنظيم الكود

تم عمل:

* تنظيف bot.py
* تقليل التكرار
* تحسين قراءة البيانات
* جعل الكود أسهل للصيانة.

---

## المرحلة 4 — لغة الإدارة

تم إضافة دعم:

```
ADMIN_LANG
```

الأدمن يستطيع اختيار:

```
ar
en
```

وهذا يؤثر على:

* أزرار الإدارة
* رسائل الجروب

بينما **العميل يختار لغته بنفسه داخل البوت**.

---

## المرحلة 5 — نقل الإعدادات إلى .env

تم نقل:

```
STORE_NAME
CLIQ_NAME
USDT_WALLET
USDT_NETWORK
SUPPORT_USERNAME
```

إلى `.env` بدل وضعها داخل الكود.

هذا يجعل المشروع مناسبًا للـ production.

---

## المرحلة 6 — Startup Validation

تم إضافة فحص عند تشغيل البوت:

إذا كانت القيم التالية ناقصة:

```
BOT_TOKEN
GROUP_ID
ADMIN_USER_IDS
```

يتوقف التشغيل مباشرة.
